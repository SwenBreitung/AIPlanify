ng new project-name - Erstellt ein neues Angular-Projekt.
ng serve - Startet einen Entwicklungsserver und öffnet die Anwendung im Browser.
ng serve --open - Startet den Entwicklungsserver und öffnet die Anwendung automatisch im Standard-Webbrowser.
ng build - Kompiliert die Anwendung in das Verzeichnis /dist.
ng build --prod - Kompiliert die Anwendung für die Produktionsumgebung mit Optimierungen.
ng test - Startet die Testlaufzeit und führt Unit-Tests aus.
ng e2e - Führt End-to-End-Tests mit Protractor aus.
ng generate component component-name - Erstellt eine neue Komponente.
ng generate directive directive-name - Erstellt eine neue Direktive.
ng generate service service-name - Erstellt einen neuen Service.
ng generate module module-name - Erstellt ein neues Modul.
ng generate pipe pipe-name - Erstellt eine neue Pipe.
ng generate guard guard-name - Erstellt einen neuen Guard.
ng generate class class-name - Erstellt eine neue Klasse.
ng generate interface interface-name - Erstellt eine neue Interface.
ng generate enum enum-name - Erstellt ein neues Enum.
ng add package-name - Fügt ein neues Paket zur Anwendung hinzu und konfiguriert es.
ng update - Aktualisiert die Angular-Anwendung und ihre Abhängigkeiten.
ng deploy - Bereitet die Anwendung für die Bereitstellung vor und veröffentlicht sie, wenn ein Bereitstellungspaket konfiguriert ist.
ng lint - Führt Linting-Tools auf Projektcode aus, um Codequalität und Konsistenz zu gewährleisten.
ng config - Ruft Konfigurationseinstellungen ab oder setzt diese.
ng help - Zeigt eine Hilfe-Seite mit allen verfügbaren Befehlen und deren Optionen an.
ng version - Zeigt die aktuelle Version von Angular CLI und die Umgebungsdetails an.
ng doc search-term - Öffnet die offizielle Angular-Dokumentation für den angegebenen Suchbegriff im Browser.
ng run - Führt eine Angular CLI-Anwendung gemäß einer angegebenen Konfiguration aus.
ng xi18n - Extrahiert i18n-Nachrichten aus dem Quellcode.
ng generate library library-name - Erstellt eine neue Bibliothek im aktuellen Angular-Projekt.
Zusätzliche Befehle
ng serve --port 4201 - Startet den Entwicklungsserver auf einem alternativen Port, hier 4201.
ng build --aot - Kompiliert die Anwendung mit Ahead-of-Time (AOT) Kompilierung für eine verbesserte Startzeit.
ng build --watch - Beobachtet Dateiänderungen und führt automatisch einen erneuten Build durch.
ng test --code-coverage - Führt Unit-Tests aus und erstellt einen Code-Coverage-Bericht.
ng generate module app-routing --flat --module=app - Erstellt ein neues Routing-Modul und fügt es dem AppModule hinzu, ohne einen neuen Ordner zu erstellen.
ng generate component components/my-component --module=app - Erstellt eine neue Komponente und fügt sie dem AppModule hinzu.
ng config cli.defaultCollection @angular-eslint/schematics - Ändert die Standard-Schematics-Sammlung, die Angular CLI verwendet, zu Angular ESLint.
ng deploy --base-href=/my/app/ - Bereitet die Anwendung für die Bereitstellung vor und aktualisiert den Base-Href-Wert.
ng update @angular/core @angular/cli - Aktualisiert Angular Core und Angular CLI auf die neueste Version.
ng add @angular/material - Fügt Angular Material zur Anwendung hinzu und führt eine Grundkonfiguration durch.
Unterbefehle
ng serve - Startet den Entwicklungsserver.
--open - Öffnet die Anwendung automatisch im Webbrowser.
--port 4201 - Startet den Server auf einem spezifischen Port, hier 4201.
--ssl - Startet den Server mit SSL, sodass die Anwendung über HTTPS verfügbar ist.
ng build - Kompiliert die Anwendung in das /dist-Verzeichnis.
--prod - Kompiliert die Anwendung für die Produktionsumgebung mit Optimierungen.
--aot - Verwendet Ahead-of-Time Kompilierung.
--source-map - Erstellt Source Maps für die Debugging-Unterstützung.
ng test - Startet die Testlaufzeit und führt Unit-Tests aus.
--watch=false - Führt Tests einmalig aus und beendet dann den Prozess (nicht im Watch-Modus).
--code-coverage - Erstellt einen Bericht über die Testabdeckung.
ng generate - Generiert oder modifiziert Dateien basierend auf einer Schematic.
component my-component - Erstellt eine neue Komponente namens 'my-component'.
service my-service - Erstellt einen neuen Service namens 'my-service'.
module my-module - Erstellt ein neues Modul namens 'my-module'.
ng update - Aktualisiert die Anwendung und ihre Abhängigkeiten.
@angular/core - Aktualisiert @angular/core und die damit verbundenen Abhängigkeiten auf die neueste Version.
--force - Erzwingt die Aktualisierung von Paketen und löst Version-Konflikte automatisch.
ng add - Fügt ein neues Paket zur Anwendung hinzu und konfiguriert es.
@angular/material - Fügt Angular Material hinzu und führt eine Grundkonfiguration durch.
ng config - Ruft Konfigurationseinstellungen ab oder setzt diese.
cli.defaultCollection @angular-eslint/schematics - Ändert die Standard-Schematics-Sammlung zu Angular ESLint.
